Overview

What is G-Code?

G-code (ISO 6983) is the numerical control programming language that tells a CNC machine where to move, how fast, and with which tool. Every machining operation — from a simple facing pass to a five-axis turbine blade — ultimately reduces to a sequence of G-code blocks that the machine controller interprets in real time.

Although CAM software generates most G-code today, understanding the underlying syntax is essential for editing post-processor output, troubleshooting alarms, and optimizing cycle times manually at the machine.

Block Structure

Anatomy of a G-Code Block

Each line (block) is a sequence of word-address pairs. A word is a letter followed by a numerical value. Multiple words can appear on one block; the controller processes them simultaneously or in a defined priority order.

N010 G00 G90 G54 X50.0 Y25.0 S2500 M03
N020 G43 H01 Z5.0
N030 G01 Z-2.0 F180.0
N040 X100.0 Y50.0
N050 G00 Z50.0
N060 M05
N070 M30

N — Sequence

Block number. Optional but helps navigation and error messages.

G — Preparatory

Defines motion type, coordinate mode, compensation, cycles.

X Y Z — Position

Target coordinates in the active work frame. Units set by G20/G21.

F — Feed Rate

mm/min (G94) or mm/rev (G95). Active until changed.

S — Spindle Speed

RPM or surface speed (with G96 constant surface speed mode).

M — Miscellaneous

Machine functions: spindle on/off, coolant, tool change, program end.

Key G-Codes

Essential G-Code Reference

G00

Rapid positioning — no cutting. Maximum axis velocity.

G01

Linear interpolation at programmed feed rate.

G02 / G03

Circular interpolation CW / CCW. Requires I, J (or K) or R.

G40 / G41 / G42

Cutter radius compensation: cancel / left / right of programmed path.

G43 / G49

Tool length offset apply / cancel. Compensates measured tool length.

G81–G89

Fixed canned cycles: drilling, boring, tapping — reduce program length.

G90 / G91

Absolute / incremental positioning mode.

G54–G59

Work coordinate offsets — each fixture has its own origin.

Validation

Program Verification

Before running a new program on the machine, always verify it through at least two layers: graphical simulation in the CAM software or a standalone G-code simulator (checks toolpath geometry and gouges), and dry run on the machine with the Z-axis offset raised so the spindle travels in air. A third layer — single-block mode — executes one block at a time, letting the operator confirm each move before it happens.

Common errors to catch: rapid moves into the part (G00 where G01 is needed), wrong work offset selection, missing tool length compensation, and feeds not appropriate for the operation.
